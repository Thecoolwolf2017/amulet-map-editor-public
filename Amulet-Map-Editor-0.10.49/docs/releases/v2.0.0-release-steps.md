# v2.0.0 Final Release Steps

## 1) Confirm Burn-In Complete
- Review `docs/releases/v2.0.0-rc1-burn-in.md`.
- Ensure no open crash/data-loss regressions.

## 2) Re-run Release Checks
- `python scripts/release_smoke_test.py`
- Manual Bedrock and Java restore workflows.

## 3) Finalize Changelog Header
- Promote `v2.0.0-rc1` notes to `v2.0.0` section in `CHANGELOG.md`.
- Keep RC notes below final section or link to release notes.

## 4) Tag and Publish (Git Repository Required)
- `git add -A`
- `git commit -m "release: v2.0.0"`
- `git tag -a v2.0.0 -m "v2.0.0"`
- `git push origin main --tags`

## 5) Build and Upload Binary-Only Asset
- Build and upload only the Windows release zip (`Amulet-<tag>-Windows-x64.zip`).
- Do not upload source distributions (`.tar.gz`) or wheels (`.whl`).
- Workflow: `.github/workflows/python-build.yml`.

## 6) Code Sign Before Distribution
- Configure signing secrets in GitHub Actions:
  - `WINDOWS_SIGN_CERT_BASE64` (base64-encoded PFX)
  - `WINDOWS_SIGN_CERT_PASSWORD`
  - Optional `WINDOWS_SIGN_TIMESTAMP_URL` (defaults to `http://timestamp.digicert.com`)
- See `docs/releases/code-signing.md`.

## 7) Release Notes
- Summarize:
  - Bedrock open crash fix
  - Backup lock handling and CURRENT repair
  - Save result UX
  - Reduced log noise
